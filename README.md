# Pen to Figma

A Figma plugin that converts [Pencil](https://pencil.evolveui.com/) (`.pen`) design files into native Figma nodes. Frames, text, auto-layout, colors, gradients, strokes, effects ... all converted. No copy-pasting screenshots.

## Why

Pencil is great for AI-driven design prototyping. Figma is great for collaboration and handoff. This bridges the two. Design fast in Pencil, push to Figma when you need to share.

## What it converts

| .pen | Figma | Status |
|------|-------|--------|
| Frames + auto-layout | Frames + auto-layout | Full support |
| Text (font, size, weight, spacing, alignment) | Text with loaded fonts | Full support (with fallback chain) |
| Rectangles, ellipses, lines | Rectangles, ellipses | Full support |
| Solid fills (`#RRGGBB`, `#RRGGBBAA`) | Solid paints | Full support |
| Gradients (linear, radial, angular) | Gradient paints | Full support |
| Strokes (solid, per-side thickness) | Strokes | Full support |
| Corner radius (uniform + per-corner) | Corner radius | Full support |
| Effects (drop shadow, inner shadow, blur) | Effects | Full support |
| Padding, gap, justify, align | Auto-layout properties | Full support |
| `fill_container` / `fit_content` sizing | `FILL` / `HUG` sizing | Full support |
| Icon fonts (Lucide, etc.) | Placeholder frame + label | Partial (no SVG import yet) |
| Image fills | Gray placeholder | Partial |
| Paths / vectors | Rectangle placeholder | Partial |
| Component instances (`ref`) | Flat copy | Not yet |

## Setup

```bash
git clone https://github.com/1shanpanta/pen-to-figma.git
cd pen-to-figma
npm install
npm run build
```

## Usage

### 1. Export your .pen data as JSON

If you have access to the Pencil MCP tools (via Claude Code), read the full node tree:

```
batch_get({ filePath: "your-file.pen", readDepth: 10 })
```

Save the output as a `.json` file.

### 2. Import the plugin into Figma

1. Open **Figma Desktop**
2. Go to **Plugins > Development > Import plugin from manifest...**
3. Select `pen-to-figma/manifest.json`

### 3. Run the plugin

1. Open any Figma file
2. **Plugins > Development > Pen to Figma**
3. Click **Load from file** and select your `.json` export, or paste the JSON directly
4. Click **Import to Figma**
5. All screens appear on the canvas

## How fonts work

The plugin scans all text nodes in the `.pen` data and pre-loads every unique font before creating any nodes. It tries multiple style name variations per weight (e.g., "SemiBold", "Semi Bold", "DemiBold") because Figma font style names vary across families.

If a font isn't available on your machine, it falls back to **Inter Regular**.

Fonts used in the Pencil designs need to be installed locally or available through your Figma plan. Most Google Fonts (Inter, Space Grotesk, JetBrains Mono, IBM Plex Mono, Playfair Display) are available in Figma by default.

## Project structure

```
pen-to-figma/
  manifest.json          # Figma plugin config
  package.json           # Build dependencies
  tsconfig.json          # TypeScript config
  src/
    code.ts              # Plugin entry point (UI messaging, orchestration)
    converter.ts         # .pen node -> Figma node conversion
    colors.ts            # Hex parsing, gradient transform math
    fonts.ts             # Font loading with fallback chains
    types.ts             # .pen file type definitions
    ui.html              # Plugin UI (file picker + paste area)
  dist/
    code.js              # Built plugin (generated by esbuild)
```

## Development

Watch mode for live rebuilds:

```bash
npm run watch
```

Then reload the plugin in Figma (**Plugins > Development > Pen to Figma**) after each change.

## Limitations

- **Icon fonts** render as placeholder frames with the icon name as a label. Swap them with real icons from a Figma icon library (e.g., Lucide Icons community file).
- **Image fills** become gray placeholders. Drop in the real images manually.
- **Component instances** (`ref` nodes) are flattened into regular frames. No linked components.
- **Gradient transforms** use a simplified rotation-to-matrix conversion. Complex gradient positioning may be slightly off.
